Requisitos elicitados

Introdução​

Este documento consolida todos os requisitos funcionais e não funcionais elicitados pelas técnicas de Análise Documental e Observação.

Metodologia​

As tabela apresentam os requisitos funcionais e os não funcionais, com colunas para identificação (RFxx ou RNFxx), descrição do requisito e rastreabilidade (ADxx para análise documental ou OBxx para observação, combinando origens semelhantes).1 2

Legendas:​

  • RFxx: Requisito Funcional n° xx.
  • RNFxx: Requisito Não Funcional n° xx.
  • ADxx: Análise Documental n° xx.2
  • OBxx: Observação n° xx.1

Requisitos Não Funcionais​

IdentificaçãoDescriçãoRastreabilidade
RNF01O sistema deve processar solicitações de agendamento em até 3 segundos.AD16
RNF02O sistema deve manter disponibilidade mínima de 99,5% para o serviço de agendamento.AD17
RNF03A interface de agendamento deve ser intuitiva e acessível em até três toques a partir da tela inicial.AD18
RNF04Todos os dados de pagamento e localização devem ser criptografados de ponta a ponta.AD19; AD31; OB20
RNF05O aplicativo deve funcionar corretamente em Android 8 ou superior e iOS 13 ou superior.AD20; OB22
RNF06O sistema deve suportar simultaneamente até 1 milhão de usuários ativos agendando viagens.AD21
RNF07O código do módulo de agendamento deve seguir boas práticas com arquitetura modular e documentação interna.AD22
RNF08O aplicativo deve integrar-se a APIs externas de mapas e serviços de pagamento.AD23
RNF09O usuário deve poder acessar histórico e agendar viagens 24 h por dia, 7 dias por semana.AD24
RNF10O aplicativo deve estar em conformidade com a LGPD, exigindo consentimento explícito para uso de dados.AD25; OB32
RNF11O sistema deve identificar paradas longas ou desvios inesperados e notificar o usuário.AD28
RNF12O sistema deve gerar um código PIN que o usuário informa ao motorista antes do início da viagem.AD29
RNF13A comunicação entre passageiro e motorista deve ser anonimizada dentro do aplicativo.AD30
RNF14A interface de mensagens deve carregar em menos de 2 segundos em 95% dos casos.OB21
RNF15O sistema de rastreamento de encomendas deve retornar o status em menos de 5 segundos para 95% das consultas.OB23
RNF16A busca por veículos deve ter precisão de localização de até 200 metros de raio.OB24
RNF17O cálculo de preços e prazos de viagem deve ser concluído em menos de 2 segundos para 95% das consultas.OB25
RNF18O sistema deve seguir as diretrizes de acessibilidade WCAG 2.1 nível AA.OB26
RNF19A interface deve ser projetada para ser intuitiva e fácil de usar por pessoas idosas.OB27
RNF20O sistema deve garantir integridade e precisão dos dados, prevenindo corrupção ou perda de informações.OB28
RNF21O sistema deve passar por manutenções e atualizações regulares para corrigir bugs e melhorar desempenho.OB29
RNF22O sistema deve ser escalável para lidar com picos de tráfego sem comprometer a disponibilidade.OB30
RNF23As informações de localização no mapa devem ser atualizadas com latência máxima de 300 ms.OB31
RNF24A localização do veículo no mapa deve ser atualizada em tempo real, com atraso máximo de 5 segundos.OB33
RNF25A precisão da localização de usuário e motorista deve ter erro máximo de 10 metros.OB34
